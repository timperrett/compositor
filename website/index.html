Built for editorial teams and book makers who want source, decisions, and production handoff to remain legible.

What it does

Keep authorship clean. Make production state accountable.

Authors work in clean Markdown. Compositor derives and validates production metadata from that source, rather than turning generated state into something authors must maintain by hand.

01

Durable source identity

Identify prose that persists across revision, so a changed manuscript can still be related to its earlier production context.

02

Narrative flow and composition validation

Check the declared roles, pacing, and structural composition before the work reaches layout.

03

Deterministic proofs and reports

Generate reviewable HTML proofs and machine-readable reports from the same approved source and plans.

04

Layout-ready production packages

Prepare revisioned text and package artifacts for the next production stage, with their rationale nearby.

Working with AI skills

AI skills work inside a clear editorial contract.

Compositor gives reusable skills a dependable project surface: readable Markdown, structured plans and records, and validation that makes the next action clear. It does not turn suggestions into silent changes.

HUMAN JUDGMENT

Own the story and approvals.

Write and revise prose, set editorial direction, assess candidate art, make selections and approvals, and control final layout.

AI SKILLS

Propose structured decisions.

Read Markdown and validated records; draft Flow Plans, Composition Plans, and art records; surface pacing, coverage, and visual-storytelling questions.

COMPOSITOR

Check and carry the handoff.

Parse source, preserve stable identity, validate plans and assets, report stale inputs, and build proofs and production packages.

Skills use protocols, not hidden state. They work through readable source files and structured records rather than recreating internal production state.

Suggestions remain reviewable. Ambiguous editorial, creative, and approval choices return to people.

Validation is feedback. A failed check tells the person or skill what to address; it is never a reason to conceal or bypass the problem.

Workflow

A repeatable editorial loop, not a black box.

A private illustrated story compendium can move deliberately from source to handoff without hiding structural decisions in a document maze.

source → trace
story.md (human-authored)
  └── paragraph identities
      └── story.flow.yaml + art/briefs (skill proposals)
          └── hardcover.composition.yaml
              └── validation + proof.html
                  └── reviewed package/manifest.json
  1. 1

    Author in Markdown

    People maintain clean prose, front matter, and deliberate anchors in a readable story directory.

  2. 2

    Interpret intent

    A planning skill drafts Flow and Composition Plans for roles, energy, pacing, and layout families.

  3. 3

    Validate and inspect

    Compositor checks source coverage and constraints, then produces diagnostics and an HTML proof.

  4. 4

    Specify and render art

    Separate skills maintain art records and optional candidate files using the team’s chosen tools.

  5. 5

    Select, review, approve

    Editors and art directors make each of these explicit decisions before an asset joins production.

  6. 6

    Build for layout

    Compositor creates revisioned text and packages; the layout application owns final design.

Validation findings and editorial review can return work to a person or skill before the next deterministic build. Final styling, page separation, and typesetting remain under the control of the design system and layout application.

Artwork records

Keep creative work inspectable from prompt to approval.

An art-specification skill can maintain a local YAML record for intent, source linkage, generation direction, candidates, feedback, and selection. Compositor validates its source and asset relationships without pretending validation is a creative judgment.

  1. Specify
  2. →
  3. Generate candidates
  4. →
  5. Select
  6. →
  7. Review
  8. →
  9. Approve

A rendering skill is optional and uses the creative team’s chosen tools to create candidate files. Editors and art directors select and review them; approval copies and pins the chosen file in the registry. Compositor does not generate or revise images.

Approval is a state, not a suggestion

A candidate is not a finished asset.

01 · SELECT

Capture the candidate the team wants to discuss, with feedback where useful.

02 · REVIEW + APPROVE

Make the editorial decision visible, including rejection or supersession when the work changes direction.

03 · APPROVE

Pin the approved local asset in the registry for the production handoff.

Outputs

Artifacts that make a handoff easier to inspect.

The point is not to claim a finished book layout. It is to make the transition from manuscript to layout more legible, while preserving a traceable link to what the team approved.

Validation results and JSON reports

Machine-readable diagnostics for repeatable tooling, checks, and review queues.

HTML proofs

A tangible place for editorial and visual review before a layout handoff.

Layout-ready plain-text exports

Revision-aware text prepared for the next application in the production chain.

Revisioned delivery packages

Opener, spread, manifest, and diagnostics information held together for a clearer delivery.

Quick start

Start with the manuscript you already trust.

Typical skill session

  1. 1. A person asks a planning skill to inspect a story and draft structured plans for review.
  2. 2. The team validates the Flow and Composition Plans, then reads the package assembly guide.
  3. 3. When art is needed, separate skills manage candidates while a person selects, reviews, and approves.
  4. 4. The team builds a deterministic package and hands it to layout.
compositor init
# Add a compendium and story Markdown files.
compositor build my-compendium --format json

Use JSON reports in repeatable human or skill-assisted tooling, and open the package assembly guide when the team needs to review the result.
