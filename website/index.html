Markdown to book-production artifacts

A dependable production layer for story manuscripts.

Compositor turns approved Markdown inputs into the same inspectable production outputs every time: that is deterministic in plain language. When source changes, the work can be traced, revalidated, and handed forward with context intact.

Explore the workflow See the command line ↓

Built for editorial teams and book makers who want source, decisions, and production handoff to remain legible.

What it does

Keep authorship clean. Make production state accountable.

Authors work in clean Markdown. Compositor derives and validates production metadata from that source, rather than turning generated state into something authors must maintain by hand.

01

Durable source identity

Identify prose that persists across revision, so a changed manuscript can still be related to its earlier production context.

02

Narrative flow and composition validation

Check the declared roles, pacing, and structural composition before the work reaches layout.

03

Deterministic proofs and reports

Generate reviewable HTML proofs and machine-readable reports from the same approved source and plans.

04

Layout-ready production packages

Prepare revisioned text and package artifacts for the next production stage, with their rationale nearby.

A small team can move deliberately from source to handoff without hiding the structural decisions in a document maze.

  1. 1

    Write in Markdown

    A story directory contains clean prose, front matter, and purposeful anchors.

  2. 2

    Describe narrative intent

    A Flow Plan maps source ranges to narrative roles, energy, and pacing.

  3. 3

    Choose structural composition

    A Composition Plan connects spreads to an edition and its allowed layout families.

  4. 4

  5. 5

    Compositor produces revisioned text and package artifacts for the next stage.

Final styling, page separation, and typesetting remain under the control of the design system and layout application.

Artwork records

Keep the creative lifecycle visible.

Compositor keeps structured local records connected to the story requirements. It validates intent, source linkage, candidate relationships, and the approved-asset relationship—not the creative judgment itself.

  1. Specify
  2. →
  3. Generate candidates
  4. →
  5. Review
  6. →
  7. Select
  8. →
  9. Approve
  10. →
  11. Attach

Creative teams or their chosen image tools generate candidates. Compositor does not generate or revise the image, and selection remains distinct from approval and attachment.

Outputs

Artifacts that make a handoff easier to inspect.

The point is not to claim a finished book layout. It is to make the transition from manuscript to layout more legible, while preserving a traceable link to what the team approved.

Validation results and JSON reports

Machine-readable diagnostics for repeatable tooling, checks, and review queues.

HTML proofs

A tangible place for editorial and visual review before a layout handoff.

Layout-ready plain-text exports

Revision-aware text prepared for the next application in the production chain.

Revisioned delivery packages

Opener, spread, manifest, and diagnostics information held together for a clearer delivery.

Quick start

Start with the manuscript you already trust.

compositor init
# Add a compendium and story Markdown files.
compositor build --format json
compositor proof

Use JSON reports in repeatable tooling, and open the generated proof when the team needs to review the result.
